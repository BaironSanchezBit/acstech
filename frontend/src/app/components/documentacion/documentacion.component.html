DOCUMENTACIÓN

AUTOMAGNO CORE SYSTEM

1. INTRODUCCIÓN

1.1 Proposito del Documento

El propósito principal del sistema es la automatización y optimización de todos los procesos de la empresa. Este sistema está diseñado para integrar y gestionar de manera eficiente las actividades empresariales, abarcando desde la gestión de inventarios y ventas hasta la contabilidad. Al centralizar y automatizar estos procesos, el ACS no solo mejora la productividad y la eficiencia, sino que también facilita la toma de decisiones informadas al proporcionar información precisa y en tiempo real.
La documentación del sistema ACS se ha elaborado con el objetivo de mostrar cómo funciona el sistema en detalle. Esta guía está destinada a desarrolladores, administradores del sistema y usuarios finales, y proporciona una comprensión exhaustiva de la estructura, configuración y uso del ACS. A través de esta documentación, los usuarios podrán instalar, personalizar y utilizar el sistema de manera efectiva, asegurando que todas las operaciones empresariales se realicen de forma óptima y sin contratiempos.


1.2 Alcance del Sistema

El sistema ERP ha sido diseñado para cubrir una amplia gama de necesidades empresariales, asegurando una gestión integral y eficiente de todas las operaciones. Su alcance abarca las siguientes áreas clave:


Gestión de Inventarios: Control y seguimiento detallado de existencias, movimientos de stock, y previsión de necesidades de reabastecimiento.

Ventas: Automatización de procesos de venta y seguimiento de clientes.

Gestión de Trámites: Coordinación y seguimiento de trámites administrativos y legales necesarios para el funcionamiento de la empresa.

Área Gerencial: Herramientas de análisis y reportes para la toma de decisiones estratégicas, incluyendo dashboards y KPIs.

Área Comercial: Gestión de relaciones con clientes y análisis de oportunidades de negocio.

Procesos de Compra y Venta de Autos: Gestión integral de todo el proceso de compra y venta de vehículos, desde la adquisición hasta la venta final, incluyendo la documentación y trámites necesarios.

1.3 Audiencia del documento

Desarrolladores: Responsables de la implementación, personalización y mantenimiento del sistema ERP. Necesitan entender la estructura, los componentes técnicos y cómo extender las funcionalidades del sistema.

Administradores del Sistema: Encargados de la configuración y gestión del entorno del ERP, asegurando su correcto funcionamiento, seguridad y rendimiento.

Gerentes y Directivos: Usuarios que requieren herramientas de análisis y reportes para la toma de decisiones estratégicas. Incluye gerentes generales, financieros, comerciales y de recursos humanos.

Personal de Ventas y Comercial: Usuarios que manejan la gestión de clientes y ventas. Necesitan funcionalidades para seguir y cerrar ventas, gestionar relaciones con clientes y analizar oportunidades de negocio.

Equipo de Logística e Inventario: Encargados de la gestión de inventarios, seguimiento de existencias, movimientos de stock y previsión de necesidades de reabastecimiento.

Personal Administrativo y Legal: Usuarios que gestionan y coordinan los trámites administrativos y legales necesarios para el funcionamiento de la empresa.

Usuarios Finales: Empleados de la empresa que interactúan con el sistema ERP en sus funciones diarias, incluyendo la entrada y consulta de datos, la generación de informes y la realización de tareas operativas.

1.4 Visión General del Sistema

El sistema ACS (Automagno Core System) desarrollado con el stack MEAN (MongoDB, Express.js, Angular, Node.js) es una solución integral diseñada para la automatización y optimización de los procesos empresariales. Su objetivo principal es centralizar y gestionar de manera eficiente todas las operaciones de la empresa, mejorando la productividad y facilitando la toma de decisiones informadas. Este sistema optimiza la coordinación entre departamentos, reduce costos operativos y aumenta la satisfacción del cliente al ofrecer un servicio más ágil y preciso.

1.4.1 Objetivos del Sistema

Automatización de Procesos: Reducción de tareas manuales y repetitivas, lo que permite a los empleados centrarse en actividades de mayor valor.

Eficiencia Operativa: Mejora de la eficiencia en todas las áreas de la empresa mediante la integración y centralización de datos y procesos.

Toma de Decisiones Informadas: Provisión de información precisa y en tiempo real a través de herramientas de análisis y reportes.

Escalabilidad y Flexibilidad: Capacidad de adaptarse y crecer con las necesidades cambiantes de la empresa.

Mejora en la Gestión de Recursos: Optimización de la utilización de recursos, incluyendo inventarios, personal y financieros.

1.5 Arquitectura del Sistema

El sistema se basa en la arquitectura MEAN, lo que proporciona una plataforma robusta, escalable y de alto rendimiento:

MongoDB: Base de datos NoSQL que ofrece flexibilidad y escalabilidad para manejar grandes volúmenes de datos.

Express.js: Framework web para Node.js que facilita la creación de APIs y la gestión de solicitudes HTTP.

Angular: Framework frontend que permite la creación de interfaces de usuario dinámicas y responsivas.

Node.js: Entorno de ejecución de JavaScript en el servidor que proporciona un modelo de I/O no bloqueante y eficiente.

En conclusión, este sistema es una herramienta poderosa diseñada para transformar y optimizar la forma en que la empresa gestiona sus operaciones, asegurando un flujo de trabajo más eficiente y una mayor capacidad para responder a las demandas del mercado.

2. REQUISITOS DEL SISTEMA

2.1 Requisitos Funcionales

Gestión de Inventarios:

• Control y seguimiento de existencias.
• Vista general de stock, incluyendo vehículos vendidos y disponibles.

Descarga de Contratos:

• Generación y descarga de contratos para la compra o venta de vehículos.

Control de Accesorios del Vehículo:

• Registro y seguimiento de accesorios incluidos con los vehículos.

Gestión de Trámites:

• Coordinación de trámites para traspaso de vehículos a nombre del cliente o de la empresa.
• Base de datos de tramitadores, incluyendo honorarios y costos de trámites.

Seguimiento de Créditos Aprobados:

• Registro y seguimiento de créditos aprobados a clientes.

Registro de Vehículos y Clientes:

• Registro detallado de vehículos y clientes en el sistema.

Órdenes de Trabajo:

• Creación y gestión de órdenes de trabajo para necesidades de mantenimiento y servicios de los vehículos.

Cotizaciones:

• Generación rápida de cotizaciones basadas en el inventario registrado.

Gestión Comercial:

• Gestión de relaciones con proveedores, bancos y otros agentes comerciales.

Generación de Actas:

• Generación de actas de entrega y actas de recepción de vehículos.

2.2 Requisitos No Funcionales 2

Rendimiento:

• Respuesta rápida a las solicitudes de los usuarios.
• Capacidad para manejar grandes volúmenes de datos sin pérdida de rendimiento.

Seguridad:

• Autenticación y autorización robustas.
• Protección de datos sensibles y cumplimiento de normativas de privacidad.
• Encriptación de datos en tránsito y en reposo.

Escalabilidad:

• Capacidad para adaptarse al crecimiento en el número de usuarios y volumen de datos.
• Arquitectura modular para facilitar la adición de nuevas funcionalidades.

Disponibilidad:

• Alta disponibilidad del sistema con mínimos tiempos de inactividad.
• Mecanismos de recuperación ante fallos.

Usabilidad:

• Interfaces de usuario intuitivas y fáciles de usar.
• Documentación y soporte para usuarios finales y administradores.

4. Configuración del Entorno

• Requisitos del Hardware y Software: No se requiere ningún requisito específico para este sistema.

• Guía de Instalación:

• Instalación Node.JS: Node

• Instalación Angular.JS: Angular

• Instalación Git Bash: Git

• Instalación MongoDB Compass: Mongo

• Dirigirse a Github ACS y clonarlo.

git clone https://github.com/DevopsAutomagno/ACS.git

• Abrir el Editor de Texto preferido, abrir la consola y en cada carpeta (ACS y backend) instalar dependencias (npm install –force).

npm install --force

• Descargar variables de entorno alojadas en el drive, área de programación ACS/Variables de Entorno.

• Configuración de MongoDB COMPASS: Diríjase a la carpeta backend y busque un archivo que se llame config.env, al abrir el archivo copie el value y péguelo en la conexión del mongodb compass.

7. Mantenimiento y Soporte

• Procedimientos de Mantenimiento:

1. Realizar cambios o mejoras en el sistema, compilar ACS, si hay cambios en el backend, comprimirlo en .zip.

2. Ingresar a la consola del sistema alojado en EC2 (AWS)

ssh -i "acsv3.pem" ubuntu@ec2-174-129-181-0.compute-1.amazonaws.com

3. Hacer kill a pm2 server

4. Enviar archivos desde la ubicación de desarrollo al EC2 /home/ubuntu/, luego moverlo de /var/www/html/.

5. Ir a backend y iniciar el servidor pm2 start server.js

• Gestión de Incidencias:

La gestión de incidencias o reportes, se realiza en MONDAY donde se gestiona cada problema, incidencia o cambio.
